## LOG-001 — User::schools() BadMethodCallException [KRİTİK] ✅ UYGULANACAK

### Sorun
`BaseSchoolController::validateSchoolAccess()` → `$user->schools()` çağrısı
Docker/Runtime ortamında method çalışmıyor → tüm okul route'ları 500 veriyor.

### Düzeltme
`validateSchoolAccess()` içinde `user->schools()` yerine doğrudan School sorgusu kullan:
```php
$hasAccess = \App\Models\School\School::where('id', $schoolId)
    ->where('tenant_id', $user->tenant_id)
    ->exists();
```

### Etkilenen Dosya
- `app/Http/Controllers/Schools/BaseSchoolController.php`

## LOG-002 — subscriptionHistory null first() [YÜKSEK] ✅ UYGULANACAK

### Sorun
`BaseTenantController::tenant()` → `$this->user()->tenants()->firstOrFail()`
`tenants()` = `hasMany(Tenant, 'owner_user_id')` — Eğer user tenant_id üzerinden bağlıysa (owner değilse), boş dönebiliyor.
`subscriptionHistory` içinde `paginatedResponse()` null paginator'la çağrılabiliyor.

### Düzeltme
`BaseTenantController::tenant()` metodunu `user->tenant_id` üzerinden düzelt:
```php
protected function tenant(): Tenant
{
    $tenantId = $this->user()->tenant_id;
    if (!$tenantId) {
        abort(403, 'Tenant bulunamadı.');
    }
    return Tenant::findOrFail($tenantId);
}
```

### Etkilenen Dosya
- `app/Http/Controllers/Tenant/BaseTenantController.php`

## BE-001 — Okul Silme Koruması [YÜKSEK] ✅ UYGULANACAK

### Gereksinim
Bir okulda herhangi bir sınıf oluşturulup akademik yıl girilmişse VE o sınıfa
herhangi bir öğrenci kaydı (child) yapılmışsa → okul silinemez (403/422).
Akademik yıl da değiştirilemez.

### Düzeltme
`SchoolService::delete()` içinde kontrol:
```php
$hasEnrollments = $school->children()->exists() || $school->classes()->whereHas('children')->exists();
if ($hasEnrollments) {
    throw new \Exception('Öğrenci kaydı olan okul silinemez.', 422);
}
```

### Etkilenen Dosya
- `app/Services/SchoolService.php`
